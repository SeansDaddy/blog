认证透传

  • • 原系统 Session Cookie → Gateway 解析提取身份
  • • Gateway 自签平台 Token，不造新认证体系
  • • DataProxy 层 Token→Cookie 转换，对原系统完全透明

多租隔离

  • • 企业级：不同企业路由到独立 Agent 实例
  • • 用户级：dmScope: per-channel-peer 隔离会话
  • • 新增用户零感知，无需管理员介入

三层并发控制

  • • L1：Redis 令牌桶（业务层）
  • • L2：Gateway rateLimit（网关层）
  • • L3：OpenClaw 沙箱配额（执行层）
