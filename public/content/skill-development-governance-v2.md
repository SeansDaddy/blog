# 智能体 Skill 开发与执行治理规范（v2.0）

> **来源**：飞书文档  
> **日期**：2026-04-21  
> **版本**：2.0  
> **标签**：AI Agent · 规范标准 · 治理框架

---

本文档为 LLM Agent 体系下的 Skill（技能/插件）开发与执行建立了一套完整的治理标准，确保 Skill 在沙箱环境中安全、可靠、可审计地运行。

## 设计原则

| 原则 | 说明 |
|------|------|
| 最小权限 | Skill 仅获得完成任务所必需的最小资源与权限 |
| 防御纵深 | 多层安全防护，单一防线被突破不影响整体安全 |
| 可观测性优先 | 所有执行过程必须有完整的日志与链路追踪 |
| 契约驱动 | 输入输出强 Schema 校验，保证上下游兼容性 |
| 开发者友好 | 本地开发环境与生产环境完全一致 |

## 资源限制与执行环境

### 超时控制

| 超时类型 | 默认值 | 处理方式 |
|----------|--------|----------|
| 执行超时 | 30 秒 | 超时后发送 SIGTERM，5 秒后仍未退出则 SIGKILL |
| 空闲超时 | 10 秒 | 连续无 stdout/stderr 输出视为卡死，强制终止 |
| 调度等待超时 | 60 秒 | 超过等待时间返回 QUEUE_TIMEOUT 错误 |

### CPU 与内存配额

| 资源类型 | 软限制 | 硬限制 | 超限处理 |
|----------|--------|--------|----------|
| CPU | 0.5 核 | 1.0 核 | 超硬限时 throttle，不终止 |
| 内存 | 128 MiB | 256 MiB | 超硬限立即 OOM Kill |
| 任务并发数 | - | 5 个/租户 | 超限排队或返回 RATE_LIMITED |

### 网络访问控制

| 规则 | 说明 |
|------|------|
| 出站网络 | 默认完全禁止，仅允许白名单内的内部服务 |
| 入站监听 | 绝对禁止启动任何 TCP/UDP 端口监听 |
| DNS 解析 | 仅解析内部域名白名单，或完全禁用 |
| ICMP | 禁止 ping 等 ICMP 通信 |

白名单申请流程：Skill 需在元数据中声明 `network_access` 字段，经安全评审后方可开通。

### 沙箱隔离等级

| 等级 | 技术实现 | 适用场景 |
|------|----------|----------|
| Level 1 | 容器（Docker + Seccomp） | 内部可信 Skill |
| Level 2 | gVisor | 一般生产 Skill |
| Level 3 | Firecracker microVM | 高风险/不可信 Skill |

## 接口协议与数据契约

### 强制 Schema 校验

每个 Skill 必须提供 `input_schema` 和 `output_schema`，平台在执行前后进行强制校验：

- **输入校验**：失败返回 INVALID_INPUT，不进入沙箱
- **输出校验**：失败计入平台健康度监控，触发告警
- **额外字段**：不允许未声明的额外字段（`additionalProperties: false`）

### 向后兼容性

| 变更类型 | 兼容性 | 版本号变更要求 |
|----------|--------|----------------|
| 增加可选字段 | ✅ 兼容 | 次版本号 +1 |
| 删除可选字段 | ✅ 兼容 | 次版本号 +1 |
| 增加必填字段 | ❌ 不兼容 | 主版本号 +1 |
| 修改字段类型 | ❌ 不兼容 | 主版本号 +1 |
| 删除必填字段 | ❌ 不兼容 | 主版本号 +1 |

### Summary Mode

Skill 必须支持 `mode` 参数以节省 LLM 上下文 Token：

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| full | 返回完整结构化数据 | 后续需要详细处理的场景 |
| summary | 仅返回核心结论字段 | 直接展示给用户或 LLM 摘要 |

### 统一错误码

| 错误码前缀 | 类别 | 示例 |
|------------|------|------|
| E10XX | 参数/输入校验 | E1001: MISSING_REQUIRED_FIELD |
| E11XX | 参数/输入校验 | E1101: INVALID_FORMAT |
| E20XX | 外部依赖异常 | E2001: API_TIMEOUT |
| E21XX | 外部依赖异常 | E2101: API_AUTH_FAILED |
| E30XX | 沙箱资源限制 | E3001: OOM_KILLED |
| E31XX | 沙箱资源限制 | E3101: DISK_QUOTA_EXCEEDED |
| E40XX | 业务逻辑失败 | E4001: CITY_NOT_FOUND |
| E50XX | 平台内部错误 | E5001: SANDBOX_START_FAILED |

## 日志与审计规范

### 日志输出流分离

| 输出流 | 用途 | 处理方式 |
|--------|------|----------|
| stdout | 业务数据/返回结果（JSON 格式） | 返回给调用方 |
| stderr | 运行时日志、调试信息 | 存入日志系统，仅内部可见 |

### stderr 结构化日志格式

每行一条 JSON，必须包含以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| timestamp | ✅ | ISO 8601 格式，UTC 时区 |
| level | ✅ | INFO / WARN / ERROR |
| skill_id | ✅ | Skill 唯一标识 |
| task_id | ✅ | 本次执行的任务 ID |
| trace_id | ✅ | 全链路追踪 ID |
| message | ✅ | 人类可读的日志信息 |

### 敏感信息脱敏

**禁止记录的内容：**
- 密钥、Token、密码、API Key
- 用户个人身份信息（邮箱、手机号、身份证号）
- 内部 IP 地址、数据库连接串

**脱敏示例：**

| 数据类型 | 脱敏方式 |
|----------|----------|
| API Key | `sk-a1****b2c3` |
| 手机号 | `138****5678` |
| 邮箱 | `t***@example.com` |

## 可用性设计

### 幂等性设计

Skill 应设计为幂等：相同输入重复执行不会产生副作用。若涉及外部写操作，必须使用唯一 `idempotency_key` 防重。

### 重试策略

| 错误类型 | 是否重试 | 最大重试次数 | 退避策略 |
|----------|----------|--------------|----------|
| 超时 | ✅ | 2 次 | 指数退避（1s, 2s） |
| OOM | ❌ | 0 | - |
| 网络临时错误 | ✅ | 2 次 | 指数退避 |
| 业务逻辑错误 | ❌ | 0 | - |
| 参数校验失败 | ❌ | 0 | - |

### 断路器模式

平台侧对每个 Skill 的失败率进行监控：

| 阈值 | 动作 |
|------|------|
| 1 分钟内失败率 > 50% | 进入半开状态，限流 10% |
| 1 分钟内失败率 > 80% | 熔断，所有请求快速失败 |
| 熔断后 30 秒 | 尝试放行 1 个请求探测恢复 |

## 安全性设计

### 沙箱隔离

| 隔离项 | 措施 |
|----------|------|
| 文件系统 | 只读根文件系统，仅 /tmp 可写 |
| 系统调用 | Seccomp 白名单，禁止 ptrace、mount、reboot |
| Capabilities | 全部 drop，仅保留必要能力 |
| /proc 和 /sys | 只读挂载或完全屏蔽 |

### 密钥管理

| 规则 | 说明 |
|------|------|
| 禁止硬编码 | 密钥不得出现在代码或配置文件中 |
| 注入方式 | 通过环境变量或 Secrets 挂载注入 |
| 日志遮蔽 | 环境变量在日志中自动脱敏 |
| 临时凭证 | 外部服务访问使用 STS 临时 Token |

### 依赖安全

- **SBOM 生成**：提交时自动生成软件物料清单
- **CVE 扫描**：使用 Trivy / Snyk 扫描已知漏洞
- **阻断策略**：存在 Critical / High 级别漏洞时禁止合并

## Skill 开发核心 Checklist

1. **Schema 校验**：提供 input_schema 和 output_schema，`additionalProperties: false`
2. **Description**：遵循「动词 + 对象 + 条件 + 返回」结构
3. **Summary Mode**：支持 `mode` 参数（full/summary）
4. **标准响应**：stdout 输出固定 JSON 结构，失败返回错误码
5. **日志分离**：stdout=业务数据，stderr=JSON 结构化日志
6. **敏感脱敏**：禁止记录密钥、Token、PII 信息
7. **幂等设计**：外部写操作使用 idempotency_key
8. **重试策略**：超时/网络错误可重试，OOM/业务错误不重试
9. **fallback**：配置降级行为（return_cached/return_default/fail）
10. **禁止硬编码密钥**：通过环境变量注入
