② 多租隔离

用户级隔离

动态路由

能力边界 · 三层流控
③ 能力边界
SELECT ONLY
AI 平台定位为纯读 + 建议系统：

· 所有 API 调用均为 SELECT 查询，不执行任何写操作
· 诊断结论展示给用户，不直接生成工单
· 不下发控制指令
· DataProxy 层做数据过滤，防止越权数据返回
④ 三层并发控制
L1
Redis 令牌桶
业务层限速，防用户级别突发
L2
Gateway rateLimit
网关层流控，平台注入配置
L3
OpenClaw 沙箱配额
执行层资源隔离
三层各司其职，任一层失效不会导致整体失效。
